# 제작이력 2 — 목록 + 캘린더 UI

## 시기
분할매수/매도 지원 이후

## 컨셉
거래(Trade) = 하나의 영역. 매수·매도 내역을 테이블 rows로 나열.  
뷰 모드 3종: 목록 / 카드 / 캘린더.

---

## 뷰 모드

### ≡ 목록 (TradeHistory)
- 계좌별로 그룹핑, 계좌 헤더 표시
- 각 거래 영역: 헤더 + 내역 테이블
  - 헤더: 종목명, 종목코드, 매수 N건·매도 N건, 상태(보유중 N일 / 완료 수익률 + 손익금액), 수정·삭제
  - 테이블 컬럼: 구분(매수/매도 배지) | 날짜 | 단가 | 수량 | 금액
  - 코멘트 있으면 테이블 하단에 💬 표시
- 정렬: 첫 번째 entry 날짜 기준 내림차순 (최신 거래가 위)

### ▦ 카드 (TradeCard)
- 기존 카드 UI 유지, 보유중 → 완료 순서로 출력

### 📅 캘린더 (TradeCalendar)
- 월별 그리드 (월~일 시작)
- 매도완료 거래만 집계, 마지막 매도 entry 날짜 기준
- 날짜 셀: 일별 손익 합계 + 거래 건수
- 월 상단: 해당 월 전체 손익 합계
- 날짜 클릭 → 해당 일 거래내역 상세 펼침 (매수·매도 rows 전체)
- 손익 색상: 양수 빨강, 음수 파랑 (한국 관례)

---

## DB 구조
```
Trade
  id, accountId, symbol, symbolCode, comment, createdAt, updatedAt

BuyEntry
  id, tradeId, date, price, quantity, createdAt

SellEntry
  id, tradeId, date, price, quantity, createdAt
```
- holdingDays, profitAmount, profitRate 는 DB 저장 안 함 → `enrichTrade()`로 조회 시 계산

## 주요 컴포넌트

## 새 거래 정책
- 새 거래 입력 = 항상 새 Trade 영역 생성 (같은 계좌+종목이어도 자동 합산 안 함)
- 기존 거래에 내역 추가 → 수정 모달에서 entry 추가
