## 1. 캘린더 뷰 (TradeCalendar)

### 만든 이유
목록/카드 뷰는 개별 거래를 보기엔 좋지만, 날짜별로 얼마를 벌었는지 한눈에 파악하기 어려움.
매도 완료 기준으로 일별 손익을 달력에 표시하고 싶다는 요구에서 출발.

### 핵심 데이터 구조 — `byDate`
```typescript
// 완료 거래를 마지막 매도일 기준으로 날짜별로 묶은 Map
byDate = {
  "2026-04-15": [거래A, 거래B],
  "2026-03-22": [거래C],
  ...
}
```
- API를 추가로 호출하지 않고, `trades` prop(이미 로드된 전체 목록)을 JS에서 가공
- 모든 날짜 집계(일별, 월별)는 이 Map을 기반으로 처리

### 데이터 흐름
```
page.tsx: /api/trades → trades[] (전체)
                ↓ prop으로 전달
         TradeCalendar
                ↓ useMemo
         byDate (완료 거래 → 날짜별 Map)
                ↓
   ┌────────────┴─────────────┐
캘린더 그리드             6개월 요약
(해당 월 날짜 필터)      (prefix × 6)
```

### 6개월 실현손익 요약
- `"2026-04-"` 같은 prefix로 `byDate`를 필터링 → 추가 API/DB 접근 없음
- 6개월 = 같은 작업 6번 반복
- 각 월 클릭 시 해당 월 캘린더로 바로 이동
- 거래 없는 달은 `-` 표시

### 날짜 클릭 → 상세 펼침
- 선택한 날짜의 거래 목록을 캘린더 아래에 인라인으로 펼침
- 매수/매도 내역 전체 표시 (TradeHistory와 동일한 테이블 구조)
- 수정·삭제 버튼 포함

## 2. 주요 UX 개선

### 매수 없이 매도만 저장 가능
- **변경 전**: 매수 내역 필수, 없으면 저장 차단
- **변경 후**: 매도만 있어도 저장 가능 (나중에 수정으로 매수 추가)
- **적용 위치**: `TradeModal.tsx` 프론트 유효성 검사 + `api/trades/route.ts` 서버 유효성 검사 동시 수정
- **이유**: 매도 알림이 먼저 오는 경우 탭을 왔다갔다해야 하는 불편함 해소

### 불완전한 row 자동 제외
- **변경 전**: 날짜·단가·수량 중 하나라도 비면 에러로 저장 차단
- **변경 후**: 완전히 채워진 row만 저장, 빈 row는 조용히 무시
- **이유**: 카톡 탭에서 데이터를 붙여넣는 중간 상태에서 저장을 막는 게 불편

### 매도 섹션을 매수 위로
- 폼 순서: 매도 → 매수 순으로 변경
- 이유: 매도 알림부터 입력하는 실제 사용 패턴 반영

### 일부 매도 시 헤더에 손익 표시
- 보유중이지만 일부 매도가 있으면 헤더에 추가 표시
  ```
  보유중 4일  잔여 1주  |  일부매도  +500원  +2.5%
  ```
- 이유: 부분 매도 결과를 목록에서 바로 확인하고 싶다는 요구

### 필터 계좌 → 새 거래 모달 연동
- 목록 상단에서 특정 계좌 필터 선택 후 "새 거래" 클릭 시 해당 계좌 자동 선택
- `TradeModal`에 `defaultAccountId` prop 추가, `page.tsx`에서 `accountFilter` 전달

## 3. 버그 수정

### `crypto.randomUUID` HTTP 환경 오류
- **원인**: `crypto.randomUUID()`는 HTTPS 또는 localhost에서만 동작 (브라우저 보안 정책)
- **해결**: `lib/utils.ts`에 `uuid()` 헬퍼 추가 — HTTPS면 네이티브, HTTP면 `Math.random()` 폴백

### `formatRate(null)` TypeError
- **원인**: 매수 없이 매도만 있는 거래 → `avgBuyPrice = 0` → `profitRate = Infinity 또는 NaN`
- **해결 1**: `enrichTrade`에서 `avgBuyPrice > 0` 조건 추가
- **해결 2**: `formatRate`에 null/Infinity 가드 추가 (`!isFinite(n)` → `'-'` 반환)
