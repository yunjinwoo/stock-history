### 2-6. 캘린더 집계 기준 변경 (PR #23)

**변경 파일**: `components/TradeCalendar.tsx`

#### 이전 방식
- 완료된 종목(`isCompleted === true`)의 **마지막 매도일** 기준으로만 집계
- 보유중 종목, 부분 매도는 캘린더에 표시되지 않음

#### 변경 후 방식
- 매수/매도 **각 항목의 실제 날짜** 기준으로 전체 집계
- 보유중 종목의 부분 매도도 해당 날짜에 표시

#### 핵심 타입 및 계산 로직
```ts
type SellEntry = { trade: Trade; price: number; quantity: number }
type BuyEntry  = { trade: Trade; price: number; quantity: number }
type DayData   = { sells: SellEntry[]; buys: BuyEntry[] }

function sellProfit(s: SellEntry) {
  return (s.price - s.trade.avgBuyPrice) * s.quantity
}
```

#### 캘린더 셀 표시
- 매도가 있는 날: 당일 실현손익 `(매도가 - 평균매수가) × 수량` 합산
- 거래가 있는 날: `매수 N건 · 매도 N건` 레이블
- 날짜 클릭 시: 해당 날짜에 활동이 있는 종목별로 그룹화하여 상세 표시

#### 총손익 vs 캘린더 합계 차이
| 항목 | 대상 | 집계 방식 |
|------|------|-----------|
| SummaryBar 총손익 | 완료 종목만 | `trade.profitAmount` 합산 |
| 캘린더 6개월 | 전체 (보유중 포함) | 매도 항목별 `sellProfit()` 합산 |

→ 보유중 종목의 부분 매도분이 캘린더에는 포함되지만 총손익에는 미포함 — **의도적인 차이**

### 2-7. 모바일 하단 탭바 (PR #24)

**변경 파일**: `components/BottomTabBar.tsx`(신규), `app/layout.tsx`, `app/page.tsx`, `app/coins/page.tsx`

#### 구조
```
[주식] [코인] [메모] [더보기]
                      ↓
               종목관리 / 계좌관리
```

- `sm:hidden` — 모바일(640px 미만)에서만 표시, 데스크탑은 기존 헤더 유지
- 현재 경로(`usePathname()`)에 따라 활성 탭 파란색 표시
- **더보기** 탭: 탭바 위로 팝업 메뉴 표시 (종목관리 / 계좌관리)
- `app/layout.tsx`에 추가해 모든 페이지에 자동 적용
- 콘텐츠가 탭바에 가리지 않도록 `<div className="h-14 sm:hidden" />` 스페이서 추가
- 헤더의 코인/주식 링크 → `hidden sm:inline-flex` (탭바로 대체)

## 5. 패턴 정리

- **Tailwind 반응형**: `hidden sm:flex`(모바일 숨김, PC 표시) / `sm:hidden`(모바일만 표시)를 조합해 화면 크기별 UI 분기
- **브랜치 혼선 예방**: 작업 시작 전 `git checkout main && git pull`로 base 확인 필수
- **PR 충돌 원인**: 같은 파일을 건드리는 두 브랜치가 동시에 열려 있으면 머지 순서와 관계없이 충돌 가능
- **전역 레이아웃 활용**: 모든 페이지에 공통 적용할 UI(탭바 등)는 `app/layout.tsx`에 한 번만 추가
