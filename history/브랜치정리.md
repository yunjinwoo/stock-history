# 브랜치 정리 이력

`feature/jinwoo/*` 형태의 로컬·원격 브랜치를 일괄 삭제한 작업 기록.

---

## 1단계 — 브랜치 현황 확인

```bash
git branch -a | grep "feature/jinwoo"
```

**용도**: 로컬과 원격에 남아 있는 `feature/jinwoo/*` 브랜치 전체 목록 확인

**결과**:
- 로컬 브랜치 18개 확인
  - `calendar-by-trade-date`, `coin-edit-ux`, `docs`, `docs-update`, `memo-category-count`, `memo-category-count-v2`, `memo-enhancements`, `memo-per-page`, `mobile-bottom-tabbar`, `mobile-responsive`, `mobile-ux-improvements`, `status-badge-color`, `stock-master`, `stock-master-filter`, `stock-memo`(현재 브랜치), `stock-tag`, `trade-images`, `ux-improvements`
- 원격 트래킹 브랜치 13개 확인 (`remotes/origin/feature/jinwoo/...`)

---

## 2단계 — main 브랜치로 전환

```bash
git checkout main
```

**용도**: 현재 브랜치(`feature/jinwoo/stock-memo`)가 삭제 대상이므로, 삭제 전 main으로 이동

**결과**: `main` 브랜치로 전환 성공

---

## 3단계 — 로컬 브랜치 일괄 삭제

```bash
git branch | grep "feature/jinwoo" | xargs git branch -D
```

**용도**: 로컬의 `feature/jinwoo/*` 브랜치 전체를 강제 삭제 (`-D` = 머지 여부 무시하고 강제 삭제)

**결과**: 18개 로컬 브랜치 전체 삭제 성공
```
Deleted branch feature/jinwoo/calendar-by-trade-date (was 1a7e08f).
Deleted branch feature/jinwoo/coin-edit-ux (was 209355a).
Deleted branch feature/jinwoo/docs (was 08d67d8).
... (18개 전체)
```

---

## 4단계 — 원격 브랜치 삭제 시도 (git push)

```bash
git branch -r | grep "feature/jinwoo" | sed 's|remotes/origin/||' | xargs -I{} git push origin --delete {}
```

**용도**: 로컬 원격 트래킹 목록을 파싱해 `git push origin --delete`로 원격 브랜치 삭제 시도

**결과**: 전부 실패
```
error: unable to delete 'origin/feature/jinwoo/coin-edit-ux': remote ref does not exist
...
```
→ `git branch -r` 목록이 stale(오래된 캐시) 상태여서 실제 GitHub에 없는 ref를 삭제하려 했기 때문

---

## 5단계 — GitHub 실제 브랜치 확인

```bash
gh api repos/yunjinwoo/stock-history/branches --paginate --jq '.[].name' | grep "feature/jinwoo"
```

**용도**: GitHub API를 직접 호출해 원격에 실제로 존재하는 브랜치 목록 확인

**결과**: 13개 브랜치가 실제로 남아 있음을 확인
```
feature/jinwoo/coin-edit-ux
feature/jinwoo/docs
feature/jinwoo/docs-update
feature/jinwoo/fix-comma-input
feature/jinwoo/memo-category-count
feature/jinwoo/memo-category-count-v2
feature/jinwoo/memo-enhancements
feature/jinwoo/memo-per-page
feature/jinwoo/mobile-bottom-tabbar
feature/jinwoo/mobile-responsive
feature/jinwoo/status-badge-color
feature/jinwoo/stock-master-filter
feature/jinwoo/ux-improvements
```

---

## 6단계 — GitHub API로 원격 브랜치 일괄 삭제

```bash
for branch in feature/jinwoo/coin-edit-ux feature/jinwoo/docs ...; do
  gh api repos/yunjinwoo/stock-history/git/refs/heads/$branch -X DELETE
done
```

**용도**: `git push --delete`가 실패한 상황에서 GitHub REST API(`DELETE /git/refs/heads/{branch}`)를 직접 호출해 원격 브랜치 삭제

**결과**: 13개 전체 삭제 성공
```
Deleted: feature/jinwoo/coin-edit-ux
Deleted: feature/jinwoo/docs
... (13개 전체)
```

---

## 7단계 — 로컬 원격 트래킹 ref 정리

```bash
git fetch --prune
```

**용도**: GitHub에서 삭제된 브랜치에 해당하는 로컬 원격 트래킹 ref(`remotes/origin/feature/jinwoo/*`) 제거. `--prune` 옵션이 원격에 없는 ref를 자동으로 삭제

**결과**: 13개 stale ref 정리 완료, 최종 브랜치 목록:
```
* main
  remotes/origin/HEAD -> origin/main
  remotes/origin/main
```

## 교훈

- `git push origin --delete <branch>`: git이 알고 있는 트래킹 정보가 stale하면 실패할 수 있음
- 확실한 방법: `gh api .../git/refs/heads/<branch> -X DELETE`로 GitHub API 직접 호출
- `git fetch --prune`: 원격 삭제 후 반드시 실행해 로컬 트래킹 ref도 동기화
