### 2-1. 메모 카드 배경색 + 핀 기본값 (PR #51)

**배경색 기준**
```
평점 1점  → bg-gray-100  (리뷰 제외 표시)
```

**완료 거래 토글**: 계좌별로 완료 건수 "완료 N 표시/숨기기" 버튼  
- 기본: 숨김 (`Set<string>` 상태, 빈 셋 = 전부 숨김)
- 클릭 시 해당 계좌만 토글

**종목 필터**: 보유 종목 요약 영역에서 종목명 클릭 → 해당 종목만 필터링

- 헤더에 활성 필터 칩 + ✕ 버튼

**가격 차트 (TradeChart)**: 기존 타임라인 바 → recharts ScatterChart로 교체
- X축: 거래 날짜 (timestamp), Y축: 단가
- 매수=파란 점, 매도=주황 점
- 평균 매수가 기준선 (dashed)
- 중복 날짜 tick key 오류 방지: `Array.from(new Set([...buyData, ...sellData].map(d => d.x))).sort()`

```tsx

  <Scatter name="매도" data={sellData} fill="#f97316" />
```

### 2-4. 종목별 보기 뷰 (PR #54)

**SymbolHistory 컴포넌트 신규 작성**

계좌 구분 없이 같은 종목끼리 합산해 하나의 블록으로 표시:
- 모든 계좌의 매수·매도 내역을 날짜순 통합
- 테이블에 "계좌" 컬럼 추가
- 가격 차트 포함 (전체 항목 합산 기준)

**헤더 표시 규칙**
```
보유중 → 전체 잔여수량 + 가중평균 매수가 + 보유일 + (완료 있으면) 실현손익
완료   → 완료 N건 + 합산 손익 + 평균 수익률
```

**정렬**: 보유중 종목 먼저, 각 그룹 내에서 최근 거래일 순

**토글 버튼**: 상단 뷰 선택에 "종목" 버튼 추가 (≡ 계좌별 / 종목 / ▦ 카드 / 📅)

### 2-5. 코인 목록 정렬 수정 (PR #55)

```ts
// 변경 전
[...trades].sort((a, b) => lastEntryDate(b).localeCompare(lastEntryDate(a)))

// 변경 후
[...trades].sort((a, b) => {
  if (a.isCompleted !== b.isCompleted) return a.isCompleted ? 1 : -1
  return lastEntryDate(b).localeCompare(lastEntryDate(a))
})
```

## 3. 주요 트러블슈팅

### recharts 관련 오류 3건 (PR #53)
| 오류 | 원인 | 해결 |
|------|------|------|
| `toLocaleString of undefined` | formatKRW가 undefined 수신 | `v != null && !isNaN(v)` 가드 추가 |
| CustomTooltip undefined price | hover 시 payload.price undefined | `if (!point || price == null) return null` |
| Duplicate key `tick-label-...` | 같은 날짜의 buy/sell 점이 동일 tick 생성 | `new Set()`으로 중복 제거 후 sort |

### GitHub Actions SSH i/o timeout
- 원인: `sudo ufw enable` 실행 시 22번 포트 규칙 없이 활성화 → 신규 SSH 연결 차단
- 구조: iwinv 클라우드 방화벽(1차) → OS UFW(2차), UFW에 22 규칙 없어서 차단됨
- 해결: `sudo ufw allow 22/tcp && sudo ufw reload` 또는 `sudo ufw disable` (클라우드 방화벽만 사용)

### PR #53 머지 충돌
- 원인: main에 PR #52 머지 후 PR #53의 TradeHistory.tsx가 충돌
- 해결: `git rebase origin/main` → 충돌 구간 수동 해결 (feature 브랜치 버전 유지) → `git push --force-with-lease`

## 5. 패턴 정리

- **Set 기반 토글 상태**: `Set<string>`으로 per-item 토글 관리 — 배열 indexOf 탐색 대신 O(1) has/add/delete
- **가중평균 계산**: `Σ(price × qty) / Σqty` — 여러 계좌의 포지션을 합산할 때도 동일 공식 적용
- **recharts XAxis tick 중복 방지**: 두 Scatter 시리즈가 같은 날짜를 가지면 tick key 충돌 → `new Set()` 후 sort로 해결
- **PATCH로 항목 추가**: 새 거래 POST 대신 기존 거래 PATCH — buyEntries/sellEntries 배열 전체를 교체하는 방식이므로 기존 항목 유지하면서 추가 가능
