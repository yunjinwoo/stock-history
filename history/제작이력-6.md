### 2-1. 모바일 UX 개선 (PR #26)

**변경 파일**: `app/memos/page.tsx`, `app/page.tsx`, `components/TradeCard.tsx`

- **메모 카드 레이아웃**: 콘텐츠와 버튼을 가로 나란히(`flex-row`) → 세로 쌓기(`space-y-2`)로 변경

- **카드 뷰 이미지**: `TradeCard`에 `TradeImageZone` 추가 — 카드 뷰에서도 이미지 업로드/조회/삭제 가능

### 2-3. 종목 태그 기능 (PR #28)

**변경 파일**: `prisma/schema.prisma`, `app/api/stock-master/[id]/route.ts`, `app/stock-master/page.tsx`, `app/page.tsx`

#### 설계 결정
- 태그는 **종목관리(StockMaster)에서 중앙 관리** → 같은 종목의 모든 거래에 자동 적용
- 개별 거래마다 태그를 달 필요 없음
- 메모에는 이미 `category` 필드가 있어 태그와 역할 중복 → 종목에만 적용
- 태그 없는 종목은 태그 필터 시 **숨김** (의도된 동작)

#### DB 변경
```prisma
model StockMaster {
  tags  String?  // 쉼표 구분 문자열: "반도체,대형주,배당"
}
```
- SQLite에 배열 타입 없어 쉼표 구분 문자열로 저장
- 파싱: `tags.split(',').filter(Boolean)`

#### 종목관리 UI
- 각 종목 행에 태그 pill 표시
- `+ 태그` 버튼 클릭 → 인라인 입력창 → Enter로 추가
- pill의 `×` 클릭으로 태그 삭제
- 검색 시 태그 내용도 포함해서 필터링

```tsx
// 태그 추가
async function handleAddTag(item, tag) {
  const newTags = [...parseTags(item.tags), tag.trim()].join(',')
  await apiFetch(`/api/stock-master/${item.id}`, {
    method: 'PATCH',
    body: JSON.stringify({ tags: newTags }),
  })
}
```

#### 주식 메인 페이지 태그 필터
- 종목 마스터 전체를 fetch해서 `symbol → tags[]` 맵 생성
- 등록된 모든 태그를 **드롭다운 버튼** 하나로 접어서 표시

##### 드롭다운 UX
- 버튼: `태그 ▼` / 선택 시 `태그 (N) ▲` + 파란색 강조
- 드롭다운 내부: 태그 검색창 + 체크박스 멀티셀렉트
- 멀티셀렉트 OR 조건: 선택한 태그 중 하나라도 가진 종목 표시
- 전체 해제 버튼, 바깥 클릭 시 닫힘
- 검색 행 안에 배치 (별도 행 없음)

```tsx
// 태그 필터 적용
const displayTrades = tagFilters.length > 0
  ? trades.filter(t => tagFilters.some(tag => symbolTagMap[t.symbol]?.includes(tag)))
  : trades
```

## 3. 트러블슈팅

### `prisma generate` 순서 오류 (반복 패턴)
- **증상**: `Unknown argument 'tags'` / `column does not exist` 에러
- **원인**: 스키마 변경 후 dev 서버가 켜진 상태에서 `prisma generate` 실행 → DLL 잠금
- **올바른 순서**:
  1. dev 서버 종료
  2. `npx prisma db push`
  3. `npx prisma generate`
  4. dev 서버 재시작

### main 브랜치 직접 커밋 (실수)
- **원인**: 브랜치 전환 후 현재 브랜치 확인 없이 커밋
- **해결**: 커밋 직후 새 브랜치 생성 → local main `reset --hard origin/main` → 정상화
- **예방**: 커밋 전 `git branch --show-current` 확인

## 5. 패턴 정리

- **Prisma 스키마 변경 시 반드시**: dev 서버 종료 → `db push` → `generate` → 재시작
- **태그 저장 방식**: SQLite에선 배열 대신 쉼표 구분 문자열이 심플하고 충분함
- **태그 중앙 관리**: 종목 단위로 관리하면 거래마다 중복 입력 없이 일관성 유지
- **드롭다운 UX 패턴**: 버튼 1개로 숨기기 + 내부 검색 + 체크박스 = 항목이 많아도 깔끔
- **`fixed inset-0`으로 backdrop**: 드롭다운 바깥 클릭 감지에 z-index 레이어 활용
