## 2. 기능 상세

### 2-1. 설계 결정

- 별도 테이블 없이 **기존 `Memo` 테이블에 `symbol String?` 필드 추가**
  - 기존 메모 기능(평점, 분류, 이미지) 그대로 재활용
  - 종목관리 페이지에서 작성 → 메모 페이지에서도 조회 가능
- 종목관리에서 작성 시 기본값: `category = '종목'`, `showOnMain = false`, `showOnCoin = false`
  - 종목 분석 메모는 메인 핀 표시가 기본 꺼짐 (필요 시 메모 페이지에서 토글)

### 2-2. DB 변경

```prisma
model Memo {
  // 기존 필드 유지 ...
  symbol  String?  // 연결 종목명 (선택, StockMaster.symbol과 문자열 연결)
}
```

### 2-3. API 변경

**변경 파일**: `app/api/memos/route.ts`, `app/api/memos/[id]/route.ts`

- `GET /api/memos?symbol=삼성전자` — symbol 쿼리 파라미터로 필터링 지원
- `POST /api/memos` — `symbol`, `showOnMain`, `showOnCoin` 수용
- `PATCH /api/memos/[id]` — `symbol` 수용

### 2-4. 종목관리 UI

**변경 파일**: `app/stock-master/page.tsx`

- 각 종목 카드에 **메모 N건 ▼** 버튼 추가
  - 메모 없으면 점선 스타일, 있으면 초록 강조
- 클릭 시 해당 종목 메모 목록 펼침
  - 메모 내용 + 등록일 + × 삭제 버튼
  - 각 메모에 `MemoImageZone` — 이미지 업로드/조회/삭제
- `+ 메모 추가` 버튼 → 인라인 textarea → Enter 추가 / Shift+Enter 줄바꿈 / Escape 취소
- 추가 시 자동으로 `category: '종목'` 설정

```tsx
// 추가 시
await apiFetch('/api/memos', {
  method: 'POST',
  body: JSON.stringify({
    content: trimmed,
    symbol: item.symbol,
    category: '종목',
    showOnMain: false,
    showOnCoin: false,
  }),
})
```

### 2-5. 메모 페이지 UI

**변경 파일**: `app/memos/page.tsx`

- 종목 연결된 메모에 **초록 종목 뱃지** 표시 (분류 뱃지 앞에 위치)
- 분류 필터 아래 **종목 필터 행** 추가 (종목 메모가 1건 이상일 때만 표시)
  - 전체 / 종목명 버튼으로 필터링
  - 분류 필터와 AND 조건으로 동시 적용

```tsx
const filtered = memos.filter(m => {
  if (categoryFilter && m.category !== categoryFilter) return false
  if (symbolFilter && m.symbol !== symbolFilter) return false
  return true
})
```

## 4. 패턴 정리

- **기존 엔티티 확장 vs 신규 테이블**: 기능이 유사하면 필드 하나 추가가 중복 구현보다 낫다
- **symbol 연결 방식**: StockMaster와 FK 없이 symbol 문자열로 연결 (Trade와 동일한 패턴)
- **쓰기 위치 / 읽기 위치 분리**: 종목관리에서 쓰고 메모 페이지에서도 읽는 구조 — 단일 테이블로 자연스럽게 해결
