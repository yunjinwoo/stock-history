## 1. 기획 및 설계

### 요구사항
- 주식 거래내역과 별도로 코인 전용 페이지 추가
- 목록 형태는 주식과 유사하되 **계좌 개념 없음**
- 새 거래 버튼은 목록 페이지 내에 배치 (메인 페이지 아님)

### 주식과의 차이점
| 항목 | 주식 | 코인 |
|------|------|------|
| 계좌 | 필수 | 없음 |
| 수량 단위 | Int (정수) | Float (소수점 가능, 예: 0.5 BTC) |
| 그룹핑 | 계좌별 | 없음 (전체 목록) |
| 입력 방식 | 카톡 붙여넣기 | 거래소 체결내역 붙여넣기 |

```prisma
  id          String           @id
  symbol      String
  comment     String?

  updatedAt   String

  price     Float
  quantity  Float     // 소수점 지원

}

model CoinSellEntry { ... }  // CoinBuyEntry와 동일 구조
```

## 4. 거래소 체결내역 파서

### 입력 형식 (거래소 체결내역)
```
2026.04.28 10:32
PRL
KRW
매수
162.61498880PRL
540.0KRW
87,812KRW
43.90KRW
87,856KRW
2026.04.28 10:32
```

### 파싱 전략
- 각 거래 항목은 **정확히 10줄** 구조 (빈 줄 없이 연속 붙여넣기 가능)
- 줄 0: 체결시간 → ISO 날짜로 변환 (`YYYY.MM.DD HH:mm` → `YYYY-MM-DDTHH:mm:00`)
- 줄 1: 코인명
- 줄 3: 종류 (매수/매도/입금/출금) → **매수·매도만 처리, 입금·출금 자동 무시**
- 줄 4: 거래수량 → 숫자 부분만 추출 (`162.61498880PRL` → `162.61498880`)
- 줄 5: 거래단가 → 숫자 부분만 추출, 콤마 제거 (`2,930.0KRW` → `2930.0`)
- `start += 10`으로 다음 항목 이동

### 결과 처리 UX
1. 파싱 후 종목별로 그룹화해서 표시
2. **전체 저장** 버튼 → 모든 종목 순차 저장 후 모달 닫힘
3. **저장** 버튼 (종목별) → 해당 종목만 즉시 저장, ✓ 표시
4. **수정 후 저장** → 직접입력 탭으로 이동해서 편집 가능

## 5. 트러블슈팅

### 500 에러 — `prisma.coinTrade is not a function`
- **원인**: 스키마에 CoinTrade 모델 추가 후 Prisma 클라이언트 미재생성
- **잘못된 시도**: `npx prisma generate --no-engine`
  - 이 플래그는 Prisma Accelerate(클라우드) 전용 클라이언트 생성
  - SQLite와 호환 불가 → `URL must start with prisma://` 에러 발생
- **올바른 해결 순서**:
  1. dev 서버 종료 (Ctrl+C) — Windows DLL 파일 잠금 해제
  2. `npx prisma generate`
  3. `npm run dev`

### Windows에서 `prisma generate` EPERM 에러
- **원인**: dev 서버가 실행 중이면 query_engine DLL이 잠겨있어 덮어쓰기 불가
- **해결**: 반드시 dev 서버 종료 후 generate 실행

**Q1. 기능 요청**

> 새거래 버튼은 목록 페이지 내에 추가하고 싶어"

→ 아래 순서로 구현

**1. DB 모델 설계** (`prisma/schema.prisma`)
- `CoinTrade` 테이블: id, symbol, comment, createdAt, updatedAt
- `CoinBuyEntry` 테이블: id, tradeId(FK), date, price, quantity(Float), createdAt
- `CoinSellEntry` 테이블: 위와 동일 구조
- 관계: CoinTrade → CoinBuyEntry/CoinSellEntry (1:N, onDelete: Cascade)

- ⚠️ dev 서버 종료 후 `npx prisma generate` 로 타입 재생성 필수

## 7. 주요 설계 결정

- **`enrichCoinTrade()`를 별도로 작성한 이유**: Prisma 타입이 `CoinTrade`와 `Trade`가 다르고, Float 수량 처리가 필요해서 `enrichTrade()`를 재사용하지 않고 별도 함수 작성
- **수량 표시 `formatQty()`**: 정수면 그대로(`10`), 소수면 불필요한 0 제거(`0.07976556`)
- **새 거래 버튼 위치**: 메인 페이지가 아닌 코인 페이지 헤더에 배치 (사용자 요구)
