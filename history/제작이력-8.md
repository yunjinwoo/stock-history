## 2. 기능 상세

### 2-1. 메모 알림 날짜 (PR #33)

**DB 변경**: `Memo.alertDate String?` 추가

```prisma
model Memo {
  alertDate String?  // YYYY-MM-DD, 현재일 이상이면 주식 페이지에 핀 노출
}
```

**동작 원리**: `alertDate <= today` (YYYY-MM-DD 문자열 사전순 비교 = 날짜 비교)

**변경 파일**
- `app/api/memos/route.ts`, `[id]/route.ts` — alertDate 읽기/쓰기
- `app/memos/page.tsx` — AlertDatePicker 컴포넌트, 알림 배지(활성=오렌지/미도래=회색), 🔔 필터 버튼
- `app/page.tsx` — Memo 인터페이스에 alertDate 추가
- `components/MemoStrip.tsx` — alertDate <= today 메모를 오렌지 배경으로 노출

```tsx
// MemoStrip: 알림 활성 메모는 showOnMain 여부와 무관하게 주식 페이지에 표시
const visible = memos.filter(m => {
  const pinned = page === 'stock' ? m.showOnMain : m.showOnCoin
  const alerted = page === 'stock' && !!m.alertDate && m.alertDate <= today
  return pinned || alerted
})
```

### 2-3. 메모 달력 뷰 + 작성일 수정 (PR #35)

**변경 파일**
- `components/MemoCalendar.tsx` — 신규 컴포넌트
- `app/memos/page.tsx` — 뷰 모드 토글, 달력 연동, 작성일 수정
- `app/api/memos/[id]/route.ts` — createdAt PATCH 지원

**MemoCalendar 기능**
- 최근 6개월 건수 바 (클릭 시 해당 월로 이동)
- 날짜 기준 토글: **작성일** / **🔔 알림일** — 달력에 표시되는 메모가 달라짐
- 날짜 클릭 시 목록 필터링 (알림일 모드에서는 오렌지 강조)
- 카테고리별 컬러 점(●)으로 해당 날짜 메모 분류 시각화

```tsx
// byDate: dateMode에 따라 다른 필드로 인덱싱
const byDate = useMemo(() => {
  const map: Record<string, MemoItem[]> = {}
  memos.forEach(m => {
    const raw = dateMode === 'alertDate' ? m.alertDate : m.createdAt
    if (!raw) return
    ;(map[raw.slice(0, 10)] ??= []).push(m)
  })
  return map
}, [memos, dateMode])
```

**작성일 수정**
- 수정 폼 하단에 날짜 input 추가
- 수정 저장 시 `createdAt` 전달 → API에서 DB 업데이트
- 메모를 몰아서 작성해도 날짜를 분산시킬 수 있게 하기 위한 설계

```tsx
// PATCH에 createdAt 추가
...(body.createdAt !== undefined && { createdAt: body.createdAt }),
```

## 3. 질문 흐름

**Q1. 알림 기능 방식**
> "메모에 알림이란 필드를 추가하고 일자를 등록하면 현재일이 등록된 일자를 지났으면 주식핀 처럼 주식 페이지에 노출되게 할수있어??"

→ `alertDate String?` 단일 필드로 구현. YYYY-MM-DD 문자열 비교로 날짜 판별.

**Q2. 시장 분류 ETF 문제**
> "이제보니 ETF 도 코스피 코스닥이 있는데..."

→ "코스피 ETF / 코스닥 ETF"로 세분화 가능하나 일단 현행 유지 결정.
추후 필요 시 옵션 추가로 대응 가능한 구조.

**Q3. 달력 출력 날짜 기준**
> "달력에 출력하는 일자를 변경가능하게 할까??"

→ 컴포넌트 내부에 `dateMode: 'createdAt' | 'alertDate'` 상태로 관리.
콜백에 mode를 함께 전달해 부모의 필터 로직도 동일한 필드로 적용.

**Q4. 달력에서 같은 날 메모 몰림 문제**
> "메모를 몰아서 작성하면 같은일에 올라가서 조금 안좋은거 같아서..."

→ `createdAt` 수정 허용. 개인 매매일지에서 사후 백데이팅은 자연스러운 패턴.
별도 "표시일" 필드를 추가하는 것보다 단순하고 직관적.

## 4. 패턴 정리

- **날짜 비교**: SQLite String 필드에 YYYY-MM-DD 저장 시 문자열 사전순 = 날짜순 — 별도 파싱 없이 `<=` 비교 가능
- **달력 컴포넌트 재사용**: TradeCalendar와 동일한 그리드 구조, 데이터 모델만 교체
- **createdAt 수정 허용**: 감사 로그가 필요 없는 개인 앱에서는 논리적 날짜 보정이 UX상 더 낫다
