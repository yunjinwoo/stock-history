## 2. 기능 상세

### 2-1. 복기(TradeTimeline) 뷰 신규 (`components/TradeTimeline.tsx`)

**배경**: 완료된 거래를 시간순으로 복기하고 싶은데, 기존 카드/종목/표 뷰는 계좌·종목별로 묶여 있어서 "최근에 뭘 잘했고 뭘 못했는지" 훑어보기 불편함.

**레이아웃**: 좌측 필터 패널 + 우측 4주 컬럼 (PC 화면 좌우 폭을 넓게 사용)
```
grid grid-cols-[160px_1fr] gap-4   // 좌: 필터, 우: 주별 히스토리
grid grid-cols-4 gap-3             // 우측 내부: 4주 컬럼
max-w-[1600px]                     // 큰 화면 여백 최소화
```

**좌측 필터 (다중 선택 가능, 조합 필터링)**
- 전체 / 익절 / 손절 (단일 선택)
- 코스피 / 코스닥 / ETF (체크박스, 복수 선택)
- 수익률 구간 7단계 (체크박스, 색상 점 표시) — 아래 색상 로직과 동일한 구간

**주차 네비게이션**: `‹ 이전주` / `다음주 ›` 버튼으로 4주 윈도우를 한 주씩 이동. `weekOffset=0`이 이번 주가 가장 오른쪽 컬럼.

**수익률 기준 색상 구분** (좌측 테두리 + 손익 텍스트)
```ts
const RATE_TIERS = [
  { id: 'red-700',  test: r => r >= 20,  ... },  // +20% 이상 — 진한 빨강
  { id: 'red-500',  test: r => r >= 10,  ... },  // +10~20%
  { id: 'red-300',  test: r => r > 0,    ... },  // 0~10%
  { id: 'gray-300', test: r => r === 0,  ... },  // 0%
  { id: 'blue-300', test: r => r > -10,  ... },  // 0~-10%
  { id: 'blue-500', test: r => r > -20,  ... },  // -10~-20%
  { id: 'blue-700', test: () => true,    ... },  // -20% 이하 — 진한 파랑
]
```
기존에는 익절/손절 두 색(red-400/blue-400)만 썼는데, 손익 크기가 안 드러나서 구간별 진하기로 세분화. 익절/손절 필터와는 별개 축(수익률 구간 필터)으로 추가.

**카드 헤더 순서**: 계좌 · 날짜 · 보유일 · 수익률을 종목명 바로 아래 한 줄로 통합
```
{accountLabel} · {MM-DD} · 보유 {N}일 · {rate}
```
계좌명 표시 시 `nickname || \`${broker} ${accountNumber}\`` 로 통일 — 기존에 `broker · nickname`을 그대로 붙여서 nickname이 broker와 같을 때 "키움 · 키움"처럼 중복 표시되던 문제 해결.

**상세보기 토글**: 카드마다 "▼ 상세" / "▲ 접기" 버튼 (수정/삭제 옆) — 클릭 시 그 카드 안에서만 가격 차트(`TradeChart`) + 매수/매도 상세 테이블 + 이미지 첨부(`TradeImageZone`)가 펼쳐짐. 완료된 거래라 목표가/손절가/예상매도가 시뮬레이터는 제외.

> 처음엔 같은 상세보기 버튼을 표(`TradeHistory.tsx`) 뷰에도 추가했다가, 복기 페이지 전용 기능이라는 걸 확인하고 표 뷰는 원래 코드로 되돌림.

### 2-2. 코인 달력 "완료된 거래만 집계" 필터 (이전 세션에서 만든 실현손익 토글의 문구/스코프 확정)

체크박스 라벨을 "실현손익 기준 (매도 시점까지의 매수만 반영)" → **"완료된 거래만 집계"**로 변경. 동작은 기존 그대로 — 체크 시 전량매도(완료)된 종목만 집계해서 상단 실현손익 합계와 정확히 일치시킴.

### 2-3. 거래완료 코멘트 (`exitComment`)

**배경**: 기존 `Trade.comment`는 매매 진입 시점 코멘트 하나뿐. 청산 후 복기(잘한 점/아쉬운 점)를 남길 자리가 없었음. 별도 `Memo` 모델(종목 태그 자유 메모, 평점/카테고리 등)과는 목적이 달라서 재사용하지 않고 Trade에 필드를 하나 더 추가.

**스키마**
```prisma
model Trade {
  comment       String?
  exitComment   String?   // 신규
  ...
}
```

**API**: `/api/trades` POST, `/api/trades/[id]` PATCH 모두 `exitComment` 지원 (기존 `comment`와 동일한 패턴).

**TradeModal**: "코멘트" 입력란 아래에 "거래완료 코멘트" textarea 추가 (placeholder: "청산 후 복기, 잘한 점/아쉬운 점 등...").

**복기 뷰 표시**: `comment`는 회색 배경 💬, `exitComment`는 노란 배경(amber-50) 📝로 구분 표시.

## 3. 트러블슈팅

### Prisma Client 재생성 시 EPERM (Windows)
`npx prisma db push` 이후 `generate` 단계에서 쿼리 엔진 DLL rename 실패:
```
EPERM: operation not permitted, rename '...query_engine-windows.dll.node.tmp...' -> '...query_engine-windows.dll.node'
```
**원인**: 로컬에서 `npm run dev`(포트 3002)가 이미 떠 있어서 Prisma가 로드한 DLL 파일이 잠겨 있음.
**해결**: TS 타입(`index.d.ts`)은 정상적으로 갱신되므로 개발에는 지장 없음. 실제 새 필드를 읽고 쓰려면 로컬 dev 서버 재시작 필요 (DLL 자체는 스키마 컬럼 추가 정도로는 안 바뀌어도 되는 경우가 많음).

### 브라우저 자동화 테스트 시 controlled input 값 미반영
Playwright/DOM에서 `textarea.value = X` 직접 대입 후 `dispatchEvent(new Event('input'))`만 하면 React 컨트롤드 컴포넌트가 변경을 감지하지 못해 실제 저장 시 빈 값으로 나감.
**해결**: React가 패치한 네이티브 setter를 우회해서 값을 설정해야 함:
```js
const setter = Object.getOwnPropertyDescriptor(window.HTMLTextAreaElement.prototype, 'value').set
setter.call(textareaEl, '값')
textareaEl.dispatchEvent(new Event('input', { bubbles: true }))
```

## 4. 패턴 정리

- **다중 체크박스 필터 조합**: 시장구분·수익률구간 모두 `string[]` 상태 + `filters.length === 0 || filters.includes(x)` 패턴 (빈 배열 = 전체 표시)로 통일
- **수익률 구간표를 배열로 선언**: `{id, label, test, border, text, dot}` 형태로 한 곳에 정의해 색상·필터·범례가 같은 소스를 공유하도록 함 (Tailwind는 동적 클래스 조합이 안 되므로 각 tier의 클래스는 리터럴 문자열로 명시)
- **계좌 표시 통일**: `account.nickname || \`${account.broker} ${account.accountNumber}\`` 패턴을 여러 컴포넌트에서 일관되게 사용
