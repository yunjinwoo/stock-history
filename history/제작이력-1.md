# 제작이력 1 — 카드 UI

## 시기
초기 버전

## 컨셉
종목 하나 = 카드 하나. 보유중 / 매도완료 섹션으로 구분.

## 화면 구성
- 상단: SummaryBar (총 투자금, 완료 손익 등 요약)
- 보유중 카드 목록 → 매도완료 카드 목록
- 카드 1장에 종목명, 계좌, 평균단가, 수량, 손익, 수익률, 보유일 표시
- 우측 상단 수정 / 삭제 버튼

## 데이터 구조 (당시)
- Trade 테이블에 단일 매수/매도 필드
  - `buyDate`, `buyPrice`, `buyQuantity`
  - `sellDate`, `sellPrice`, `sellQuantity`
- 분할매수/분할매도 불가

## 컴포넌트
- `components/TradeCard.tsx` — 카드 1장
- `app/page.tsx` — 보유중/완료 배열로 분리해서 렌더

## 한계
- 분할매수·분할매도를 지원하지 못함
- 같은 종목을 여러 번 매수하려면 별도 거래로 입력해야 했음
- 거래 내역이 많아질수록 카드가 길게 늘어져 가독성 저하

## 변경 이유
분할매수/분할매도 지원 요구 → DB 구조 변경 (BuyEntry, SellEntry 테이블 분리)
