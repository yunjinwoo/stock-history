# 주식 매매일지 프로젝트 명세서

## 프로젝트 개요

주식 매수/매도 기록을 입력·조회하고, 종목별 보유 기간 및 손익을 한눈에 파악할 수 있는 매매일지 웹앱.

---

## 핵심 기능 요구사항

### 1. 거래 입력
- **수동 입력**: 폼을 통해 직접 입력
- **카톡 알림 붙여넣기 파싱**: 증권사 카카오톡 알림 문자를 붙여넣으면 자동 파싱
  - 지원 포맷 예시:
    ```
    [키움증권] 매수체결 삼성전자 100주 @75,000원 (2024-01-15 09:32)
    [키움증권] 매도체결 삼성전자 100주 @78,000원 (2024-01-20 14:15)
    ```
  - 파싱 항목: 거래유형(매수/매도), 종목명, 수량, 단가, 체결일시

### 2. 매매 추적 뷰 (종목별 카드)
- 매수 → 매도까지 **보유 기간** 자동 계산 (일/시간 단위)
- 손익 금액 및 수익률(%) 표시
- 상태 구분: `보유중` / `매도완료`
- 매도 미완료 종목은 상단 고정

### 3. 코멘트
- 거래별 자유 메모 입력 (최대 500자)
- 목록에서는 **30자 미리보기** + 말줄임표 표시
- 클릭 시 전체 코멘트 모달 확인

### 4. 계좌 관리
- 계좌 목록 페이지 (`/accounts`)
- 계좌 추가: 증권사명 + 계좌번호 + 별명(선택)
  - 카톡 파싱 시 신규 계좌 감지되면 자동 등록 제안
- 계좌별 메모 추가/수정/삭제 (자유 텍스트, 제한 없음)
  - 예: 전략 메모, 계좌 성격(단타/스윙/장기) 등
- 계좌 삭제 시 연결된 거래 내역 경고 표시

### 5. 목록 / 필터
- 전체 목록: 최신순 정렬
- 필터: 종목명 검색 / 기간 범위 / 상태(보유중·완료) / **계좌 선택**
- 요약 통계: 총 거래수, 총 손익, 평균 수익률, 평균 보유일 (계좌 필터 반영)

## 데이터 모델

```typescript
// Account (계좌)
interface Account {
  id: string;           // UUID
  broker: string;       // 증권사명 (예: "한국투자증권")
  accountNumber: string;// 마스킹된 계좌번호 (예: "44****16-01")
  nickname?: string;    // 별명 (예: "메인 계좌")
  memo?: string;        // 계좌 메모 (자유 텍스트)
  createdAt: string;
  updatedAt: string;
}

// Trade (거래 단위: 매수 1건 + 매도 1건이 1세트)
interface Trade {
  id: string;                  // UUID
  accountId: string;           // Account.id (FK)
  symbol: string;              // 종목명 (예: "삼성전자")
  symbolCode?: string;         // 종목코드 (예: "005930")

  // 매수
  buyDate: string;             // ISO datetime
  buyPrice: number;            // 매수 단가
  buyQuantity: number;         // 매수 수량

  // 매도 (null이면 보유중)
  sellDate?: string;           // ISO datetime
  sellPrice?: number;          // 매도 단가
  sellQuantity?: number;       // 매도 수량

  // 자동 계산 필드 (DB 저장 안 함, 조회 시 계산)
  holdingDays?: number;
  profitAmount?: number;
  profitRate?: number;

  // 메타
  comment?: string;
  createdAt: string;
  updatedAt: string;
}
```

## 카톡 알림 파싱 규칙

### 날짜 처리
- 알림 문자에 **날짜 없음** → 붙여넣기 당일 날짜(today)를 자동 세팅
- 시간이 있으면(예: `11:41`) 체결시간으로 저장, 없으면 시간은 미기재
- 저장 전 날짜·시간 수동 수정 가능

---

### 증권사별 실제 파싱 패턴

#### 한국투자증권
```
[한국투자증권 체결안내]11:41

*매매구분:현금매수체결
*종목명:비츠로테크(042370)
*체결수량:41주
*체결단가:17,000원
```
| 추출 필드 | 추출 방법 |
|-----------|-----------|
| broker | 헤더 `[한국투자증권 체결안내]` |
| time | 헤더 뒤 `HH:mm` (예: `11:41`) |
| type | `*매매구분:` 값에 `매수` 포함 → 매수 / `매도` 포함 → 매도 |
| symbol | `*종목명:` 에서 괄호 앞 텍스트 (예: `비츠로테크`) |
| symbolCode | `*종목명:` 에서 괄호 안 숫자 (예: `042370`) |
| quantity | `*체결수량:` 숫자 |
| price | `*체결단가:` 숫자 (콤마 제거) |

정규식 예시:
```js
// 종목명 + 종목코드
/\*종목명:(.+?)\((\d+)\)/
// 체결수량
/\*체결수량:(\d+)주/
// 체결단가
/\*체결단가:([\d,]+)원/
// 매매구분
/\*매매구분:[현금\s]*(매수|매도)/
// 시간 (헤더)
/체결안내\](\d{1,2}:\d{2})/
```

---

#### KB증권
```
[KB증권] 주식 체결 안내

■ 종목명: 대신정보통신 
■ 주문수량: 795주 
■ 체결금액: 1,491원 
■ 내용: 매수체결(80142963)
```
| 추출 필드 | 추출 방법 |
|-----------|-----------|
| broker | 헤더 `[KB증권]` |
| time | 없음 → 시간 미기재 |
| type | `■ 내용:` 값에 `매수` / `매도` 포함 여부 |
| symbol | `■ 종목명:` 값 (trim) |
| symbolCode | 없음 (미제공) |
| quantity | `■ 주문수량:` 숫자 |
| price | `■ 체결금액:` 숫자 → **주당 단가** (사용자 확인 완료) |

정규식 예시:
```js
// 종목명
/■ 종목명:\s*(.+)/
// 주문수량
/■ 주문수량:\s*([\d,]+)주/
// 체결금액 (단가로 처리)
/■ 체결금액:\s*([\d,]+)원/
// 매수/매도 구분
/■ 내용:\s*(매수|매도)체결/
```

### 파싱 공통 로직

```
1. 입력 텍스트에서 증권사 식별 (헤더 패턴 매칭)
2. 해당 증권사 파서 적용
3. 파싱 성공 → 입력 폼 필드에 자동 채움 → 사용자가 확인·수정 후 [저장]
4. 파싱 실패 → "인식 못한 형식" 안내, 빈 폼 유지
```

#### 미확인 증권사 (구조 패턴으로 식별 — 계좌번호 083 계열)
```
계좌명 : 윤진우
계좌번호 : 083-50-3***49
종목명 : 셀바스AI
종목코드 : 108860
체결구분 : 매수체결
체결수량 : 146주
체결단가 : 15300원
-------------------------------
주문수량 : 146주
누적체결수량 : 146주
-------------------------------
체결 내역을 확인해보세요.
```
| 추출 필드 | 추출 방법 |
|-----------|-----------|
| broker | 헤더 없음 → `계좌번호` 패턴으로 식별, broker = "미확인" |
| time | 없음 → 미기재 |
| type | `체결구분 :` 값에 `매수` / `매도` 포함 여부 |
| symbol | `종목명 :` 값 (trim) |
| symbolCode | `종목코드 :` 값 |
| quantity | `체결수량 :` 숫자 |
| price | `체결단가 :` 숫자 (원 제거) |

정규식 예시:
```js
/종목명\s*:\s*(.+)/
/종목코드\s*:\s*(\d+)/
/체결구분\s*:\s*(매수|매도)/
/체결수량\s*:\s*([\d,]+)주/
/체결단가\s*:\s*([\d,]+)원/
```

#### 키움증권
```
[키움]체결통보
대주전자재료
매도11주
평균단가128,500원
```
| 추출 필드 | 추출 방법 |
|-----------|-----------|
| broker | 헤더 `[키움]` |
| time | 없음 → 미기재 |
| type | 3번째 줄에서 `매수` / `매도` 앞부분 |
| symbol | 헤더 다음 줄 (2번째 줄 전체) |
| symbolCode | 없음 (미제공) |
| quantity | 3번째 줄 `매수N주` / `매도N주` 에서 숫자 |
| price | `평균단가` 뒤 숫자 (콤마 제거) |

정규식 예시:
```js
// 종목명: 헤더([키움]체결통보) 다음 줄
lines[1].trim()
// 매수/매도 + 수량 (공백 없이 붙어있음)
/(매수|매도)([\d,]+)주/
// 평균단가
/평균단가([\d,]+)원/
```
